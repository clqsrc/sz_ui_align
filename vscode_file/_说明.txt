

主要是放运行环境的配置，免得老忘记。

一般由两个文件组成 launch.json 和 settings.json

通常的说明只会说到前面的那个，但如果每个项目的 sdk(jdk/go/gcc这些) 版本不尽相同的话，就要在本地另外写一个 settings.json

1.换命令行/终端系统
参考 ts 下面的 settings ，里面有一个将默认的 ps 改为 cmd.exe 的方法。 当初 ts 更换似乎是为了解决 ps 对 nodejs 不兼容的一些问题吧。

2.插件版本降级 -- 升级后有可能用不了，或者是 jdk 版本不想升级这样的
vscode 插件版本降级的方法是，在已安装控件列表中点击右边的齿轮，在弹出菜单中选择安装另外一个版本

3.java
只安装微软自己出的 Debugger for Java 就可以了，它应该是会自己下载 redhat 相关的东西。不行的话手工再下载也可以。
装了这个后连运行带调试以及代码提示都有了。另外也是不用安装最新的 jdk 的，只要有 jdk 8 环境就可以了。
